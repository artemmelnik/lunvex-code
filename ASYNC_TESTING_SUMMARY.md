## 🚀 Инструкции по использованию

### 1. Быстрый запуск всех основных тестов:
```bash
./run_async_tests.sh
```

### 2. Запуск всех тестов (включая требующие доработки):
```bash
python tests/test_all_async.py --mode=pytest
```

### 3. Просмотр списка всех тестов:
```bash
python tests/test_all_async.py --mode=list
```

### 4. Запуск конкретного тестового файла:
```bash
python -m pytest tests/test_async_system.py -v
```

### 5. Запуск в режиме direct (без pytest):
```bash
python tests/test_all_async.py --mode=direct
```

## 🎨 Архитектура тестовой системы

### 1. Модульная структура
- Каждая категория тестов в отдельном файле
- Простые тесты для базовой функциональности
- Сложные тесты для интеграционных сценариев

### 2. Подходы к тестированию
- **Unit-тесты:** Изолированное тестирование инструментов
- **Интеграционные тесты:** Взаимодействие компонентов
- **Тесты производительности:** Оценка параллельной обработки
- **Тесты совместимости:** Синхронные/асинхронные интерфейсы
- **Тесты обработки ошибок:** Устойчивость системы

### 3. Инструменты
- **pytest** с поддержкой асинхронности
- **asyncio** для асинхронных тестов
- **unittest.mock** для мокинга
- **tempfile** для изолированного тестирования
