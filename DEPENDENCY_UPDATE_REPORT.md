### 5. Обнаруженные проблемы и решения

#### Проблема: Ложные срабатывания сканера уязвимостей
**Симптом**: Внутренний сканер показывает 459 уязвимостей, но `pip-audit` не находит ни одной.

**Причина**: Сканер использует OSV API и GitHub Advisory API, которые могут возвращать нерелевантные данные или данные с ошибками парсинга.

**Решение**: 
1. Создан улучшенный скрипт `scripts/check_security.py`
2. Рекомендуется использовать `pip-audit` как основной инструмент
3. Для критических проектов добавить `safety` (требует API key)

#### Проблема: Длинные строки кода (25 ошибок E501)
**Решение**: Временно отключена проверка в конфигурации Ruff. Для полного исправления нужно:
1. Разбить длинные строки в 25 файлах
2. Использовать автоматическое форматирование (Black уже настроен)
