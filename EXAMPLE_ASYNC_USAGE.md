# Пример использования асинхронного режима с субтасками

## Простой пример

```bash
# Запуск в интерактивном режиме (новый стандарт)
lunvex-code

# Вы увидите приветственное сообщение:
# ┌─────────────────────────────────────────────────────────────┐
# │                    LunVex Code v0.1.0                       │
# │                Terminal AI coding assistant                 │
# └─────────────────────────────────────────────────────────────┘
# ✓ Working in: /path/to/your/project
# ✓ Async mode with automatic subtask planning enabled
#
# >
```

## Примеры задач

### 1. Простая задача (без планирования)
```bash
lunvex-code --no-planning "Прочитай файл README.md"
```

**Что происходит:**
- Система выполняет задачу напрямую без разбиения на субтаски
- Идеально для простых, одношаговых операций

### 2. Средняя сложность (автоматическое планирование)
```bash
lunvex-code "Добавь документацию к функции calculate_total в utils.py"
```

**Что происходит:**
1. Система анализирует задачу и определяет, что она достаточно сложна
2. Создается план с субтасками:
   - **Субтаск 1**: Прочитать и понять функцию `calculate_total`
   - **Субтаск 2**: Написать документацию в формате docstring
   - **Субтаск 3**: Проверить, что документация корректна
3. Субтаски выполняются последовательно с передачей контекста

### 3. Сложная задача (параллельное выполнение)
```bash
lunvex-code "Обнови зависимости, запусти тесты и сгенерируй отчет о покрытии"
```

**Что происходит:**
1. Анализ задачи выявляет несколько независимых компонентов
2. Создается план с возможностью параллельного выполнения:
   - **Субтаск A**: Обновить зависимости (не зависит от других)
   - **Субтаск B**: Запустить тесты (не зависит от других)
   - **Субтаск C**: Сгенерировать отчет (зависит от B)
3. **Субтаски A и B выполняются параллельно**
4. После завершения B выполняется C
5. Все результаты объединяются в финальный отчет

## Практические примеры

### Пример 1: Рефакторинг кода
```bash
lunvex-code "Рефакторинг модуля user_management: раздели на отдельные классы для User, Profile и Permissions"
```

**Автоматически созданный план:**
```
📋 Task Execution Plan
Original task: Рефакторинг модуля user_management...
Number of subtasks: 4

1. analyze: Проанализировать текущую структуру user_management
   Dependencies: none
   Files: user_management.py, models.py

2. design: Спроектировать новую структуру классов
   Dependencies: analyze
   Files: user_management.py

3. implement: Реализовать новые классы User, Profile, Permissions
   Dependencies: design
   Files: user_management.py, models.py

4. test: Протестировать рефакторинг
   Dependencies: implement
   Files: test_user_management.py, user_management.py
```

### Пример 2: Создание нового функционала
```bash
lunvex-code "Добавь систему логирования в проект: создай logger.py с rotation, добавь конфигурацию и пример использования"
```

**Автоматически созданный план:**
```
📋 Task Execution Plan
Original task: Добавь систему логирования...
Number of subtasks: 5

1. research: Изучить существующую структуру проекта
   Dependencies: none
   Files: requirements.txt, setup.py, main.py

2. design: Спроектировать архитектуру логирования
   Dependencies: research
   Files: (none - создание нового)

3. implement: Реализовать logger.py с rotation
   Dependencies: design
   Files: logger.py (новый)

4. integrate: Интегрировать логирование в основной код
   Dependencies: implement
   Files: main.py, app.py, config.py

5. document: Добавить документацию и примеры
   Dependencies: integrate
   Files: README.md, logger.py, examples/
```

## Советы по использованию

### Когда использовать `--no-planning`
- Простые одношаговые задачи ("прочитай файл", "запусти тесты")
- Когда нужен быстрый результат без накладных расходов
- Для отладки или исследования

### Когда довериться автоматическому планированию
- Сложные многошаговые задачи
- Когда не уверены в лучшем подходе
- Для задач, затрагивающих multiple файлы
- При работе с большими codebases

### Оптимизация производительности
- Система автоматически выполняет независимые задачи параллельно
- Для IO-bound операций (чтение файлов, сетевые запросы) выигрыш значительный
- Для CPU-bound операций параллелизм ограничен для избежания перегрузки

## Отладка и мониторинг

### Включение verbose режима
```bash
lunvex-code --verbose "Сложная задача"
```

**Вывод включает:**
- Детали анализа сложности задачи
- Подробности создания плана
- Информацию о выполнении каждого субтаска
- Время выполнения и ресурсы

### Проверка статуса
В интерактивном режиме:
```
> /status
Current Status:
  YOLO Mode:  off
  Trust Mode: off
  Model:      deepseek-chat
  Mode:       Async with automatic subtask planning
```

## Заключение

Новая асинхронная система с автоматическим планированием задач:

1. **Упрощает работу** со сложными задачами
2. **Ускоряет выполнение** через параллелизм
3. **Улучшает качество** через структурированный подход
4. **Сохраняет гибкость** через опции командной строки
