# Система теперь использует асинхронный режим с субтасками по умолчанию

## Изменения

### 1. Новая точка входа (`lunvex_code/main.py`)
- Создан новый основной файл CLI, который использует асинхронную версию по умолчанию
- Все команды обернуты для работы с асинхронным кодом через `asyncio.run()`
- Добавлена опция `--no-planning` для отключения автоматического планирования задач

### 2. Обновление конфигурации (`pyproject.toml`)
- Точка входа изменена с `lunvex_code.cli:main` на `lunvex_code.main:main`
- Команды `lunvex-code` и `lvc` теперь запускают асинхронную версию

### 3. Улучшения в асинхронном агенте (`lunvex_code/async_agent.py`)
- Метод `run()` по умолчанию использует планирование (`use_planning=True`)
- В комментариях указано: "In async mode, planning is always used by default for better task decomposition and context management"

### 4. Улучшения в планировщике задач (`lunvex_code/task_planner.py`)
- Метод `execute_plan_async` теперь поддерживает параллельное выполнение независимых субтасков
- До 3 независимых задач могут выполняться параллельно
- Улучшена логика определения сложности задач для асинхронного контекста

### 5. Ключевые преимущества

#### Автоматическое разбиение на субтаски
- Сложные задачи автоматически разбиваются на управляемые субтаски
- Каждый субтаск имеет четкие зависимости и ожидаемый результат
- Контекст передается между субтасками

#### Параллельное выполнение
- Независимые субтаски выполняются параллельно
- Ускорение выполнения задач с независимыми компонентами
- Оптимальное использование ресурсов

#### Улучшенное управление контекстом
- Каждый субтаск получает только необходимый контекст
- Избегание переполнения контекстного окна LLM
- Более точное и focused выполнение задач

#### Гибкость настройки
- Опция `--no-planning` для простых задач
- Совместимость с существующими инструментами
- Постепенное внедрение без breaking changes

## Как использовать

### Базовое использование
```bash
# Запуск в интерактивном режиме (асинхронный с субтасками по умолчанию)
lunvex-code

# Выполнение задачи (автоматическое разбиение на субтаски)
lunvex-code "Рефакторинг всей кодовой базы"

# С отключенными проверками безопасности (только для доверенных сред)
lunvex-code --yolo "Масштабные изменения"
```

### Расширенные опции
```bash
# Отключение автоматического планирования для простых задач
lunvex-code --no-planning "Простая задача"

# Режим доверия (авто-одобрение безопасных операций)
lunvex-code --trust "Запуск тестов"

# Без анимаций (только текст)
lunvex-code --no-animation "Анализ кода"

# Специфичная модель
lunvex-code --model deepseek-coder "Написание кода"
```

### Примеры сложных задач

## Технические детали

### Как работает планирование
1. **Анализ сложности**: Система анализирует задачу на наличие индикаторов сложности
2. **Создание плана**: Генерируется план с зависимостями между субтасками
3. **Параллельное выполнение**: Независимые субтаски выполняются параллельно
4. **Управление контекстом**: Результаты передаются между зависимыми субтасками

### Критерии сложности задачи
- Множественные действия (и, затем, после, также)
- Упоминание нескольких файлов/модулей
- Четкие последовательные шаги
- Длинные описательные задачи (>25 слов)
- Сложные операции (рефакторинг, интеграция, миграция)

### Преимущества асинхронного режима
- **Производительность**: Параллельное выполнение инструментов
- **Масштабируемость**: Обработка больших задач без блокировок
- **Надежность**: Лучшая обработка ошибок и таймаутов
- **Гибкость**: Возможность добавления новых асинхронных инструментов

## Заключение

Система теперь по умолчанию использует асинхронный режим с автоматическим разбиением сложных задач на субтаски. Это обеспечивает:

1. **Лучшую производительность** через параллельное выполнение
2. **Более надежное выполнение** сложных задач через декомпозицию
3. **Улучшенное управление контекстом** для работы с большими проектами
4. **Гибкость** через опцию `--no-planning` для простых задач

Пользователи получат более быстрое и надежное выполнение сложных задач без необходимости ручного разбиения на шаги.
